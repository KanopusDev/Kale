Getting Started

Welcome to the Kale Email API! This guide will help you start sending emails through our platform in minutes.

Quick Start

  1. Create an account - Sign up for a free account to get started
  2. Configure SMTP - Add your email server settings in the dashboard
  3. Create a template - Design your first email template
  4. Generate API key - Create an API key for authentication
  5. Send your first email - Use our API to send emails

Your Personal API Endpoint

Each user gets a personalized endpoint:

https://kale.kanopus.org/{username}/{template_id}

Pro Tip: Your username becomes part of your API URL, making it easy to remember and share with your team.

Authentication

Kale uses API keys for authentication. Include your API key in the request headers or as a query parameter.

API Key Methods

1. Authorization Header (Recommended)

curl -X POST "https://kale.kanopus.org/yourname/welcome" \
  -H "Authorization: Bearer your-api-key-here" \
  -H "Content-Type: application/json" \
  -d '{"recipients": ["user@example.com"]}'

2. X-API-Key Header

curl -X POST "https://kale.kanopus.org/yourname/welcome" \
  -H "X-API-Key: your-api-key-here" \
  -H "Content-Type: application/json" \
  -d '{"recipients": ["user@example.com"]}'

3. Query Parameter

curl -X POST "https://kale.kanopus.org/yourname/welcome?api_key=your-api-key-here" \
  -H "Content-Type: application/json" \
  -d '{"recipients": ["user@example.com"]}'

Security Note: Always use HTTPS and keep your API keys secure. Never expose them in client-side code.

Sending Emails

Basic Email Sending

Endpoint

POST https://kale.kanopus.org/{username}/{template_id}

Request Body

{
  "recipients": ["user@example.com", "another@example.com"],
  "variables": {
    "name": "John Doe",
    "company": "Acme Corp",
    "custom_field": "Any value"
  }
}

Complete Example

curl -X POST "https://kale.kanopus.org/johndoe/welcome-email" \
  -H "Authorization: Bearer kale_abc123def456..." \
  -H "Content-Type: application/json" \
  -d '{
    "recipients": [
      "newuser@example.com",
      "admin@company.com"
    ],
    "variables": {
      "username": "newuser",
      "company_name": "Example Corp",
      "activation_link": "https://app.example.com/activate/xyz"
    }
  }'

Response

{
  "success": true,
  "message": "Emails sent successfully",
  "data": {
    "request_id": "req_abc123def456",
    "sent_count": 2,
    "failed_count": 0,
    "results": [
      {
        "recipient": "newuser@example.com",
        "status": "sent",
        "message_id": "msg_123456"
      },
      {
        "recipient": "admin@company.com", 
        "status": "sent",
        "message_id": "msg_789012"
      }
    ]
  }
}

Field Descriptions

Field Type Description
recipients array Array of email addresses (max 100)
variables object Template variables for personalization

Email Templates

Templates allow you to create reusable email designs with dynamic content using variables.

Template Variables

Use double curly braces to insert variables in your templates: 

<h1>Welcome {{name}}!</h1>
<p>Thank you for joining {{company_name}}.</p>
<p>Your account was created on {{signup_date}}.</p>

Get Template Information

GET https://kale.kanopus.org/{username}/{template_id}
Authorization: Bearer your-api-key

Response

{
  "success": true,
  "data": {
    "template_id": "welcome-email",
    "name": "Welcome Email",
    "description": "Welcome new users to the platform",

    "created_at": "2024-01-15T10:30:00Z",
    "updated_at": "2024-01-15T10:30:00Z"

Rate Limits

To ensure fair usage and system stability, we implement rate limits on our API.

Email Limits

Account Type Daily Limit Burst Limit
Unverified 1,000 emails/day 100 emails/5 minutes
Verified Unlimited 100 emails/5 minutes
Enterprise Unlimited Custom

API Call Limits

  • 60 requests per minute
  • 3,600 requests per hour
  • 50,000 requests per day

Rate Limit Headers

All API responses include rate limit information in the headers: 

X-RateLimit-Limit: 60
X-RateLimit-Remaining: 45
X-RateLimit-Reset: 1640995200
X-Email-Limit-Daily: 1000
X-Email-Limit-Remaining: 750

Rate Limit Exceeded: When limits are exceeded, you'll receive a 429 status code. Implement exponential backoff in your applications.

Error Handling

The Kale API uses conventional HTTP status codes and provides detailed error messages.

HTTP Status Codes

Code Status Description
200 OK Request successful
400 Bad Request Invalid request data
401 Unauthorized Invalid or missing API key
403 Forbidden Account suspended or insufficient permissions
404 Not Found Template or user not found
429 Too Many Requests Rate limit exceeded
500 Server Error Internal server error

Error Response Format

{
  "success": false,
  "error": {
    "code": "INVALID_RECIPIENTS",
    "message": "One or more recipient email addresses are invalid",
    "details": {
      "invalid_emails": ["invalid-email", "another@invalid"]
    }
  },
  "request_id": "req_abc123def456"
}

Common Error Codes

INVALID_API_KEY

The provided API key is invalid or has been revoked.

TEMPLATE_NOT_FOUND

The specified template does not exist or is not accessible.

RATE_LIMIT_EXCEEDED

You have exceeded your rate limit. Wait before making more requests.

SMTP_CONFIG_MISSING

No SMTP configuration found. Please configure your email server.

SDKs & Libraries

Official and community SDKs to integrate Kale into your applications.

JavaScript/Node.js

npm install kale-email-api
const Kale = require('kale-email-api');

const kale = new Kale({
  apiKey: 'your-api-key',
  username: 'yourname'
});

await kale.send('welcome-email', {
  recipients: ['user@example.com'],
  variables: { name: 'John' }
});

Python

pip install kale-email-api
from kale import KaleAPI

kale = KaleAPI(
    api_key='your-api-key',
    username='yourname'
)

response = kale.send(
    template_id='welcome-email',
    recipients=['user@example.com'],
    variables={'name': 'John'}
)

PHP

composer require kale/email-api
use Kale\EmailAPI;

$kale = new EmailAPI([
    'api_key' => 'your-api-key',
    'username' => 'yourname'
]);

$response = $kale->send('welcome-email', [
    'recipients' => ['user@example.com'],
    'variables' => ['name' => 'John']
]);

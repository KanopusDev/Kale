Getting Started

Welcome to the Kale Email API documentation. This guide will help you get started with sending emails through our professional API platform.

Base URL

https://kale.kanopus.org

Quick Start

  1. Create your account and get your API key
  2. Configure your SMTP settings in the dashboard
  3. Create your first email template
  4. Send your first email via API

Authentication

API Key Authentication

Kale uses API keys for authentication. Include your API key in the Authorization header:

Authorization: Bearer your-api-key-here

Getting Your API Key

You can generate and manage your API keys in the Dashboard.

Sending Emails

POST /api/v1/email/send

Send an email using a template or raw HTML content.

Request Examples

cURL

curl -X POST "https://kale.kanopus.org/api/v1/email/send" \
  -H "Authorization: Bearer your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "template_id": "welcome-email",
    "recipients": [
      {
        "email": "user@example.com",
        "name": "John Doe"
      }
    ],
    "variables": {
      "name": "John Doe",
      "company": "Acme Corp",
      "signup_date": "2025-01-15"
    },
    "subject": "Welcome to {{company}}!"
  }'

PowerShell

$headers = @{
    "Authorization" = "Bearer your-api-key"
    "Content-Type" = "application/json"
}

$body = @{
    template_id = "welcome-email"
    recipients = @(
        @{
            email = "user@example.com"
            name = "John Doe"
        }
    )
    variables = @{
        name = "John Doe"
        company = "Acme Corp"
        signup_date = "2025-01-15"
    }
    subject = "Welcome to {{company}}!"
} | ConvertTo-Json -Depth 3

Invoke-RestMethod -Uri "https://kale.kanopus.org/api/v1/email/send" `
                  -Method POST `
                  -Headers $headers `
                  -Body $body

Response Example

{
  "success": true,
  "message": "Email sent successfully",
  "data": {
    "request_id": "req_abc123def456",
    "sent_count": 1,
    "failed_count": 0,
    "delivery_time": "0.245s",
    "email_ids": ["email_xyz789"]
  }
}

Public API Endpoints

Simplified endpoints for direct email sending using your username and template ID.

POST /{username}/{template_id}

Send emails directly using your username and template ID in the URL path.

Request Examples

cURL

curl -X POST "https://kale.kanopus.org/{username}/welcome" \
  -H "Authorization: Bearer your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "recipients": ["user@example.com", "another@example.com"],
    "variables": {
      "name": "John Doe",
      "company": "Acme Corp"
    }
  }'

PowerShell

$headers = @{
    "Authorization" = "Bearer your-api-key"
    "Content-Type" = "application/json"
}

$body = @{
    recipients = @("user@example.com", "another@example.com")
    variables = @{
        name = "John Doe"
        company = "Acme Corp"
    }
} | ConvertTo-Json -Depth 3

Invoke-RestMethod -Uri "https://kale.kanopus.org/{username}/welcome-email" `
                  -Method POST `
                  -Headers $headers `
                  -Body $body

Request Body Schema

{
  "recipients": ["email1@example.com", "email2@example.com"],
  "variables": {
    "variable_name": "value",
    "another_variable": "another_value"
  }
}

Response Example

{
  "success": true,
  "message": "Emails sent successfully",
  "data": {
    "request_id": "req_abc123def456",
    "sent_count": 2,
    "failed_count": 0,
    "delivery_time": "0.189s",
    "template_used": "welcome-email",
    "username": "pradyumntandon"
  }
}

API Key Authentication

The API key can be provided in three ways:

  • • Authorization: Bearer your-api-key header
  • • X-API-Key: your-api-key header
  • • ?api_key=your-api-key query parameter

Email Templates

Create Template

POST /api/v1/templates 
curl -X POST "https://kale.kanopus.org/api/v1/templates" \
  -H "Authorization: Bearer your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Welcome Email",
    "slug": "welcome-email",
    "subject": "Welcome to {{company_name}}!",
    "html_content": "<h1>Welcome {{name}}!</h1><p>Thank you for joining {{company_name}}.</p>",
    "variables": ["name", "company_name"]
  }'

Get Template

GET /api/v1/templates/{template_id}

List Templates

GET /api/v1/templates

Webhooks

Webhooks allow you to receive real-time notifications about email delivery events.

Supported Events

  • email.sent - Email was successfully sent
  • email.delivered - Email was delivered to recipient
  • email.bounced - Email bounced
  • email.failed - Email failed to send

Webhook Payload Example

{
  "event": "email.delivered",
  "timestamp": "2025-01-15T10:30:00Z",
  "data": {
    "email_id": "email_xyz789",
    "recipient": "user@example.com",
    "template_id": "welcome-email",
    "delivery_time": "2.3s"
  }
}

Error Handling

Kale uses conventional HTTP status codes to indicate the success or failure of API requests.

200

OK

Request successful

400

Bad Request

Invalid request parameters

401

Unauthorized

Invalid or missing API key

429

Too Many Requests

Rate limit exceeded

Error Response Format

{
  "success": false,
  "error": {
    "code": "INVALID_API_KEY",
    "message": "The provided API key is invalid",
    "details": {
      "request_id": "req_abc123def456"
    }
  }
}
